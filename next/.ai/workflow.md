WORKFLOW SYSTEM:

STEP 1 — UNDERSTAND
- Analyze the problem or request
- Identify goals, constraints, and context
- Clarify ambiguous areas if needed

STEP 2 — ANALYZE
- Break the problem into components
- Identify root causes or key requirements
- Detect risks, edge cases, and dependencies

STEP 3 — PLAN
- Propose a clear approach before implementation
- Choose the simplest effective solution
- Ensure scalability and maintainability

STEP 4 — IMPLEMENT
- Write clean, structured, and modular code
- Follow best practices for the given language/framework
- Keep logic easy to follow

STEP 5 — VALIDATE
- Check correctness of logic and output
- Test for edge cases and failure scenarios
- Ensure no regressions or new issues

STEP 6 — REFINE
- Remove redundancy
- Improve readability and structure
- Optimize only if necessary

STEP 7 — FINAL AUDIT
- Confirm everything works as expected
- Ensure alignment with original goal
- Verify usability, performance, and stability

ADAPTATION:
- Adjust workflow based on task complexity  
- Skip unnecessary steps for simple tasks
- Expand analysis for complex systems

SELF-UPDATE:
- If a better workflow is identified, suggest improvements
- The workflow may evolve over time for better efficiency
